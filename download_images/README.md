# Wikimedia Quality Image Download Scripts

This directory contains small utilities for downloading and preparing
images from Wikimedia Commons category index JSON files produced by
`../contents/grab_contents.py`.


Warning: 
The following scripts are "old", and their documentations here are generated by an LLM based on the code.  I have inspected the generated documentation and believe it is free from major errors, but user discretion is advised:
- `download.py`
- `query.py`
- `prepare.py`
- `select_first.py`


## Old Scripts

- `download.py`  
  Download all images listed in Wikimedia Commons category JSON files.

  **Inputs**
  - `-s / --source`  
    Directory containing JSON files, each produced from Wikimedia
    category queries. The script expects each JSON file to have
    `query.categorymembers[*].title` entries.
  - `-d / --dest`  
    Destination directory where downloaded image files will be stored.

  **Behaviour**
  - Iterates over all JSON files in `source`, sorted by filename.
  - For each entry in `query.categorymembers`, queries the Commons API
    to resolve the direct image URL and downloads the file.
  - Saved filenames are `<json-stem>-<idx>` with the original URL
    suffix (for example `.jpg`, `.tif`, `.png`).
  - Skips entries whose stem already exists in `dest` with size
    greater than 300 KB (useful for resuming an interrupted run).
  - Uses a simple exponential back‑off (`Waiter`) when the API returns
    errors or warnings.

- `query.py`: Demonstration of how to query the Commons API
  Inspect a specific item from a category JSON file and print its
  metadata and resolved image URL.

  **Inputs**
  - `-s / --source`  
    Directory containing JSON files.
  - `-q / --query`  
    Query string in the form `<file-stem>-<index>`, where:
    - `<file-stem>` is the basename (without extension) of the JSON
      file, and
    - `<index>` is the zero‑based index into
      `query.categorymembers[*]`.

  **Behaviour**
  - Opens `<source>/<file-stem>.json`.
  - Prints the selected entry from `query.categorymembers[index]`.
  - Queries the Commons API and prints the resolved image URL.


- `prepare.py`  
  Filter, resize and convert downloaded images into JPEGs with a
  minimum resolution and acceptable aspect ratio, in parallel.

  **Inputs**
  - `-s / --source`  
    Directory with the downloaded images (for example, the output of
    `download.py`).
  - `-d / --dest`  
    Destination directory where processed `.jpg` files will be stored.
  - `--ratio-min`  
    Minimum allowed height/width ratio (default: `0.38`).
  - `--ratio-max`  
    Maximum allowed height/width ratio (default: `2.62`).
  - `--height`  
    Minimum image height in pixels (default: `1080`).
  - `--width`  
    Minimum image width in pixels (default: `1080`).
  - `-p / --process`  
    Number of worker processes for parallel processing (default: `24`).

  **Behaviour**
  - Considers only files with suffixes `.jpeg`, `.jpg`, `.tiff`,
    `.tif` in the `source` directory.
  - Discards images that:
    - have aspect ratio `h / w` outside `[ratio_min, ratio_max]`, or
    - are smaller than the requested `height`/`width`.
  - Resizes remaining images so that:
    - the shorter side matches the minimum size constraint, and
    - the longer side is scaled to preserve aspect ratio.
  - Saves images as JPEG with quality `97`.
  - Uses multiprocessing to speed up processing.

  **Example**
  ```bash
  python3 prepare.py \
    -s ./images_raw \
    -d ./images_prepared \
    --height 1080 \
    --width 1080 \
    --ratio-min 0.5 \
    --ratio-max 2.0 \
    -p 16
  ```


- `select_first.py`  
  Select the first `n` images from a directory and create symbolic
  links to them in a new directory.

  **Inputs**
  - `-s / --source`  
    Directory containing image files.
  - `-d / --dest`  
    Destination directory where symbolic links will be created.
  - `-n / --number`  
    Number of images to select (default: `20000`).

  **Behaviour**
  - Sorts files in `source` lexicographically and takes the first
    `n`.
  - For each selected file, creates a symbolic link in `dest` with the
    same filename.
  - `dest` is created if it does not exist.

  **Example**
  ```bash
  python3 select_first.py \
    -s ./images_prepared \
    -d ./images_subset \
    -n 5000
  ```

- `query-detailed-meta.py`: Query detailed metadata from condensed metadata (from `../contents/grab_contents.py`)


### Typical Workflow for Downloading and Preparing Images

1. From `wikimedia/contents/`, run `grab_contents.py` to generate
   category JSON files under `contents/`.
2. From `wikimedia/download_images/`, run `download.py` to fetch all
   image files into `images_raw/` (or another directory of your
   choice) using the JSON files from step 1.
4. Run `prepare.py` to filter and resize the raw images into
   `images_prepared/`.
5. Optionally run `select_first.py` to create a smaller subset of
   images via symbolic links.


## New Scripts

- `query-detailed-meta.py`: Query detailed metadata from condensed metadata
- `stat.py`: Compute statistics about the images
- `meta-filter.py`: Filter the images based on metadata.  It supports the following operations:
    - By licence:
        - `only-cc0`: Keep only images with CC0 licence (and those in PD)
        - `only-cc-by`: Keep only images with CC‑BY licence (and those with more permissive licences)
    - By content:
        - `only-people`, `no-people`: Keep only images with people, or keep those without people, based on keywords in `ObjectName` or `ImageDescription`
        - `include-non-photo`: Keep images that are not photos.  By default, only images that appear to be photos are kept.  This is determined by its format.  It is rather well-known that Wikimedia Commons will be unlikely to label a digital artwork saved in JPEG format as a "Quality Image". 
        - `empirical-rule-1`: Using our empirical rule for reducing the number of images with the same content (for example, many images portraying the same event).  Specifically, we make sure the processed dataset does not contain two images that meet all of the following conditions:
            - uploaded by the same user.
            - uploaded within 12 hours of each other.
            - their `ObjectName` or `ImageDescription` cannot be determined to be different enough.  Two images are considered to have different enough content if either of the following is true:
                - They both have `ObjectName` and similarity between the two `ObjectName` is low enough
                - They both have `ImageDescription` and similarity between the two `ImageDescription` is low enough
                The similarity is determined by a string following the `empirical-rule-1-`:
                    - `levenshtein`: the Levenshtein distance
                    - `clip`: the cosine similarity between CLIP text embeddings
                    - `llama`: using a chatbot provided by llama.cpp to determine whether the two strings are describing the same content.
    WARNING: the output of this script is not guaranteed to have been shuffled.  Make sure to shuffle the output before using it for training.
